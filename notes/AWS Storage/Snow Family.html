AWS Snow Family 的簡介與歷史足跡

AWS Snow Family 是一組針對大量數據遷移與邊緣計算的解決方案，適用於網絡帶寬有限或需本地化處理的情境

Snowcone (2024-11-12 AWS 宣布終止 Snowcone 服務)​

  • 8 TB
  • 很輕、防摔、防震、防塵、防水、耐高低溫
  • 可以用於資料 compute, storage, data transfer
  • 可以將資料傳到 Snowcone 後，用 DataSync 透過網路傳輸至 AWS，也可以寄回 AWS 讓 AWS 幫忙上傳
  • 在本地電腦安裝 AWS OpsHub 來管理 Snow Family 裝置
  • 可搭配 DataSync 將資料從 Snowcone 上傳到 S3
  • 可以分成
    • Snowball Edge Compute Optimized
      • 較多 CPU 資源
    • Snowball Edge Storage Optimized
      • 容量較大
  • 影片
    • 介紹
      • https://youtu.be/R-yfD-aROKM
    • 案例 - 野外影音轉碼
      • https://youtu.be/X_8LM7E_hiE?feature=shared

Snowball​

  • 80 TB (舊版)
    • AWS Snowball Edge Storage Optimized 210TB device with NVMe storage and the ability to transfer up to 1.5 gigabytes of data per second for data migration workloads, and the Snowball Edge Compute Optimized with 104 vCPUs device with full SSD 28TB NVMe storage for edge compute workloads.
  • 用於資料傳輸，可以是將資料傳進 AWS，也可以是從 AWS 傳出來
  • 大規模資料案例介紹影片
    • https://youtu.be/9uc2DSZ1wL8?feature=shared

Snowmobile (2024 年 4 月起已停止提供服務)​

  • 容量約 100 PB
  • 是乘載大量 Snowball 的大型卡車
  • 有溫控、GPS、24 小時監控攝影、甚至可以請保鑣
  • 適合 > 10 PB 的案例
  • 2024 年 4 月 AWS 為了精簡成本，取消了 Snowmobile，原因：
    • 駕駛、停車問題
    • 電力、控溫成本問題
    • 太少使用，大多數企業只用 Snowball 搬移一部分資料，很多企業上雲後不需要再動用 Snowmobile 解決
  • AWS 展示 Snowmobile 的影片
    • https://youtu.be/8vQmTZTq7nw?feature=shared

使用原則​

  • 評估自己的資料量在自己的網速傳入 AWS S3 的所需時間
    • 用網路上傳和用 Snow Family 哪個比較快? 比較穩? 成本考量?

總結​

  • 2024 年 AWS 取消多項 Snow Family 服務，現在 Snow Family 只剩 Snowball (Snowball Edge)
    • 取消原因除了成本效益，現在和 2015 年剛推出 Snowball、2016 推出 Snowmobile 以及 2020 推出 Snowcone 的時空背景不同了，網路速度大幅提升
  • 有了更快的網路後，搭配 AWS DataSync、Direct Connect、OutPost 比以前更能解決資料傳輸等問題
